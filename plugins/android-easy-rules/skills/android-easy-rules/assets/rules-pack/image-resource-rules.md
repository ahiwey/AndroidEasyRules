# Android Image / Icon Resource Rules

## 资源放置

- 新增图片前先查同类资源所在目录。
- bitmap 资源使用小写下划线命名，并带业务前缀，例如 `ic_meeting_*`、`home_*`、`sleep_*`。
- 不要使用临时名、中文名、空格或设计稿原始文件名。
- 不要把无关品牌素材、截图原图、设计稿、临时素材放进 `res`。

## PNG / WebP / XML Drawable 选择

- 复杂位图、渐变、照片类素材使用 PNG/WebP。
- 简单圆角、边框、selector、纯色背景优先 XML drawable。
- 不要为了“现代化”强行把所有图标改成 vector。
- 不要把已有项目风格完全打散。

## 使用规则

- 保持图标宽高比和透明边距。
- XML 中通过明确宽高、`scaleType`、约束和内外边距控制显示。
- 不要用拉伸填满导致变形。
- 新增可点击图片按钮时，补充可访问性描述或明确其文本语义。
- 纯装饰图不要加入误导性文案。

## 验证

- 现有图片内容替换优先检查格式、尺寸、透明边距、引用和实际截图；新增/重命名资源或引用时运行一次最小资源处理任务。
- 只有用户要求 APK、改动打包配置，或最小资源任务无法覆盖集成边界时才运行 app assemble。
- 关键页面尽量用模拟器/真机截图检查实际显示。
- 大量新增/替换图片时，检查 APK 体积和重复资源。
