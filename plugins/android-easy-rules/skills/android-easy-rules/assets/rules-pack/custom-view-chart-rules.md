# Custom View / Chart Rules

## 适用范围

- Canvas 自定义 View
- 健康图表
- 睡眠、心率、血氧、血压、步数、温度、压力等趋势图
- 自定义进度、柱状图、折线图、饼图

## 修改前

- 新增 View、重做绘制结构或改变交互时，先找同类 View 对比并延续已有风格；明确的单个间距、尺寸、颜色或裁切问题只读取目标 View 和实际使用位置。
- 先理解数据输入、单位、时间范围、空数据处理和 UI 使用位置。
- 只有涉及调用链或共享绘制逻辑时，才使用 CodeGraph 查调用方和影响范围。
- 截图已明确指出局部问题时走 Quick：一次聚焦修改后优先做 diff 和边界计算检查，不加载通用 UI/UX 流程，也不默认运行 Gradle。

## 绘制规则

- `onDraw` 中避免频繁分配对象、解析资源、创建格式化器或做重计算。
- 复杂计算应提前缓存到字段、`onSizeChanged` 或数据设置阶段。
- Canvas 变换要配对 `save/restore`。
- 文字基线、clip、padding、圆角、线宽和密度换算要清晰。
- 必须处理空数据、单点数据、极值、单位切换、文字过长、左右边界、夜间/皮肤模式和不同屏幕宽度。

## 注释和测试

- timing、animation、smoothing、scaling、business rule 相关的非直觉逻辑，用简短英文注释解释原因。
- 图表纯计算逻辑可以隔离时，优先补聚焦单元测试。
- 实际绘制效果仍以截图或人工视觉验证为准。
