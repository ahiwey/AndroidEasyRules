# Global AGENTS.md

## 全局协作规则

- 与我沟通时使用中文说明。
- 以“少读、准找、可验证”为默认工作方式：先确认任务类型和影响范围，再按最小上下文定位，不为了“了解项目”全量扫描。
- 代码修改前给出极短执行计划：准备改哪些文件、为什么、如何验证。
- 保持修改范围最小，不做顺手重构，不格式化无关文件，不覆盖用户已有改动。
- 最终回复说明改了什么、验证命令和结果、未验证风险。

## 上下文与工具

- 如果项目存在 `AGENTS.md`、`MEMORY.md` 或模块级规则，先遵守项目规则；更靠近目标文件的规则优先。
- 结构性问题优先使用项目指定的结构化工具，例如 CodeGraph；只有当前会话实际暴露对应工具时，才能声称已使用。
- 只有任务实际需要结构分析且当前会话没有 `codegraph_*` 时，才说明一次“CodeGraph 未接入”，再改用 `rg` 和精确文件读取；用户已指定文件、函数、资源或固定文本的小改无需机械声明。
- 字符串、资源名、日志、注释、固定文本等字面量查询优先使用 `rg`。
- 不要为了“可能有用”安装新插件、技能或 MCP；只有明确需要外部系统能力或用户点名时再配置。

## 模型与技能路由

- 当前模型为 `gpt-5.5` 时，根因未知、异步状态、跨文件/模块、重构、迁移和高风险平台任务使用适用的 Superpowers 流程；明确的单文件、单资源和已知改法任务走 Quick 流程，不加载 Superpowers。
- 当前模型标识以 `gpt-5.6` 开头时，默认不调用任何 `superpowers:*` 技能，即使插件已经暴露；用户在本轮明确要求使用时，以本轮要求为准。
- `gpt-5.5` 任务需要 Superpowers 但技能不可用时，只说明一次并按项目规则继续，不搜索或安装插件，也不声称已经使用。
- 不自动安装或调用 `grill-me`。探索后仍有产品决策时，只询问 1–3 个会改变实现的关键问题；用户本轮明确要求采访时再按其要求执行。

## 命令与验证

- shell 外部命令优先用 `rtk` 包装，例如 `rtk git status`、`rtk rg "keyword"`、`rtk .\gradlew.bat :app:assembleDebug`。
- PowerShell 内置命令不要直接套 `rtk`；确需过滤输出时使用 `rtk powershell -NoProfile -Command "..."`。
- 默认不进行编译校验；只有用户明确要求，或完成任务实际需要时才执行编译。
- 修改后按影响范围运行最小验证。文档和规则修改通常不需要构建，但要检查文件内容和差异。
