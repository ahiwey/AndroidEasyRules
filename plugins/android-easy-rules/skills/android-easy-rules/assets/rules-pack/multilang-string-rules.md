# Multilang String Rules

## 触发条件

- 新增、删除、重命名或批量修改 `strings.xml` key。
- 用户要求翻译、多语言同步、文案替换、品牌词替换、权限说明文案或多套 `values-*` 资源一致性。

## 执行方式

- 已知 string key 时，优先用结构化 XML 解析或复用脚本批量更新所有目标模块现有 `values*/strings.xml`；不要手工逐个 patch 大量语言文件。
- 批量更新只修改目标 key，不重排无关资源，不格式化整个 XML。
- 默认 `values/strings.xml` 与所有已有 `values-*` 下同名 key 同步；用户明确要求跳过翻译时，最终回复必须说明未同步范围。
- 可复用脚本时优先放在 `tools/` 等明确工具目录；临时脚本只用于一次性迁移，执行后删除并检查状态。

## 品牌词与旧词扫描

- 权限和设备相关文案重点检查：`戒指`、`设备`、`ring`、`device`、`watch face`、`表盘`、`錶盤`。
- 只扫描目标 key 的 value，避免 XML 标签名或其他无关文案造成误判。
- 涉及品牌词时，按当前分支品牌语义统一，不从其他品牌分支复制不适配文案。

## 默认验证

- key 覆盖率：确认目标 key 在默认和所有现有 `values-*` 目录中存在或按任务要求缺失。
- 旧词残留：确认目标 key 的 value 中没有本次要求替换掉的旧词。
- 资源编译：优先运行受影响模块资源编译或 assemble；纯规则文档修改不跑 Gradle。
- `git diff --check`：确认没有 XML 空白问题。
